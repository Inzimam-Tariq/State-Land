Adjacency Validation Guide
1. Purpose

This module validates whether Mauza adjacency relationships are correctly defined in the system. Field staff can approve or correct adjacency mappings.

2. Workflow Overview
  • Select a Tehsil from dropdown
  • System loads Mauza adjacency summary
  • Click Validate to review details
  • Check/uncheck incorrect adjacents
  • Submit with remarks
3. Status Meaning
  • Pending Review → Not yet validated
  • Verified (1/2) → First validation completed
  • Finalized → Validation locked
4. Rules
  • Each Mauza has maximum 2 validation attempts
  • After second attempt, record becomes locked
  • Remarks are mandatory for corrective actions
5. Best Practices
  • Verify adjacency using field knowledge or maps
  • Avoid marking all as correct without validation
  • Add meaningful remarks for corrections
